# Physics Visualizer

一个尝试用 AI 将中文物理题目自动转化为可交互 2D 仿真的小工具。两天内完成，覆盖范围有限。

## 启动

```bash
npm install
npm run dev          # → http://127.0.0.1:5173
npm test             # 16 tests
```

## 能做什么

- 输入一段中文物理题，自动生成可运行的 2D 仿真
- 播放、暂停、拖拽时间轴、缩放平移
- 查看速度、加速度、受力（重力/弹力/支持力/摩擦力等）的实时数值和时序图
- 调整参数滑块（质量、刚度、恢复系数等）
- 支持的基础场景：自由落体、抛体、弹簧振子、弹性/非弹性碰撞、单摆、斜面、滑轮、匀强电场/磁场、点电荷

## 不能做什么

- 复杂多体耦合（三个以上物体同时参与多种作用）
- 热力学、流体、波动、量子
- 3D 渲染
- 物理参数的自动校验——AI 生成的初始位置、质量、刚度可能不合理，结构正确不代表物理正确

## 大致思路

让 AI 输出一种结构化的物理描述（我管它叫 Graph DSL），而不是直接生成代码。DSL 描述了场景中有哪些物体、怎么连接、受什么力、初始状态——然后交给 Box2D 引擎执行。AI 和引擎通过这份协议解耦，各做各擅长的事。

AI 生成流程分了三步：先从题目提取中间表示（IR），再转成 DSL，如果校验不通过就自动修复。提示词迭代了几轮，生成通过率从最初 79% 提到 100%（DeepSeek Flash，100 道基础题测试）。

## 技术栈

React · TypeScript · Box2D-WASM · Canvas 2D · Vite · Vitest · DeepSeek API

## 目录

```
src/
├── graph/          # DSL 类型定义与校验
├── sim/box2d/      # Box2D 世界、刚体、关节、力、事件
├── sim/            # 仿真主循环
├── render/         # Canvas 渲染
├── services/       # AI 客户端、DSL 生成、提示词
├── components/     # React UI
schema/             # JSON Schema
scripts/            # 批量测试工具
examples/           # 20 个手写场景
```

## 已知问题

- AI 生成的场景物理参数可能不准确
- 部分复杂题目超出 DSL 的表达上限
- 网页未适配移动端
